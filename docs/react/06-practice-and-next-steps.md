# 06. 练习路线和下一步学什么

## 本章目标

读完这一章，你应该知道：

- 学完 React 基础后应该做哪些练习。
- 如何判断自己能不能开始做项目。
- 什么时候学 Next.js。
- 哪些 React 生态先不用急着学。

## 1. React 入门达标标准

你不需要一开始就“精通 React”。

如果你能独立写出下面这些，就算入门达标：

- 计数器。
- Tab 切换。
- 登录表单。
- 商品列表。
- 简单购物车。
- 弹窗。
- 接口请求列表。
- loading/error/empty 状态。
- 父组件传 props 给子组件。
- 子组件通过回调通知父组件。

## 2. 第一组练习：基础组件

### 练习 1：个人资料卡片

要求：

- 组件名：`UserCard`
- props：`name`、`age`、`city`
- 显示用户信息。

进阶：

- 增加 `avatarUrl` 可选字段。
- 没有头像时显示默认头像。

### 练习 2：商品卡片

要求：

- 组件名：`ProductCard`
- props：`name`、`price`、`description`
- 有一个“加入购物车”按钮。

进阶：

- `available=false` 时按钮禁用。
- 价格格式化为 `¥xx.xx`。

### 练习 3：通用 Card

要求：

- props：`title`
- 使用 `children`

示例：

```tsx
<Card title="订单信息">
  <p>订单号：1001</p>
  <button>确认</button>
</Card>
```

## 3. 第二组练习：状态和交互

### 练习 1：计数器

要求：

- 显示数字。
- +1。
- -1。
- 重置。
- 数字不能小于 0。

### 练习 2：Tab 切换

要求：

- 三个按钮：首页、订单、我的。
- 点击不同按钮显示不同内容。
- 当前按钮有 active 样式。

### 练习 3：弹窗

要求：

- 点击“打开弹窗”显示弹窗。
- 点击“关闭”隐藏弹窗。
- 弹窗内容通过 props 传入。

## 4. 第三组练习：表单

### 练习 1：登录表单

要求：

- 用户名。
- 密码。
- 提交时校验不能为空。
- 显示错误消息。

### 练习 2：注册表单

要求：

- 用户名。
- 邮箱。
- 密码。
- 确认密码。
- 密码不一致显示错误。

### 练习 3：订单表单

要求：

- 姓名。
- 手机号。
- 备注。
- 提交后显示提交的数据。

## 5. 第四组练习：列表和购物车

### 练习 1：商品列表

要求：

- 用数组渲染商品。
- 每个商品有名称、价格、按钮。
- 使用稳定 `key`。

### 练习 2：购物车

要求：

- 点击加入购物车。
- 如果商品已存在，数量 +1。
- 可以减少数量。
- 数量为 0 时移除。
- 显示总价。

### 练习 3：搜索过滤

要求：

- 输入关键词。
- 根据关键词过滤商品列表。
- 没结果时显示“暂无结果”。

## 6. 第五组练习：数据请求

### 练习 1：请求用户列表

要求：

- 页面加载时请求接口。
- 显示 loading。
- 失败显示 error。
- 空数组显示 empty。

### 练习 2：详情页

要求：

- 根据 `id` 请求详情。
- `id` 变化时重新请求。
- 注意 `useEffect` 依赖数组。

### 练习 3：提交表单

要求：

- 填表单。
- 点击提交。
- `fetch` POST 到接口。
- 提交中按钮禁用。
- 成功后显示提示。

## 7. 第六组练习：组件架构

做一个小商城页面，拆成：

```text
ShopPage
  ProductList
    ProductCard
  CartPanel
  CheckoutForm
```

要求：

- `ShopPage` 管理购物车 state。
- `ProductList` 展示商品。
- `ProductCard` 点击按钮通知父组件。
- `CartPanel` 展示购物车。
- `CheckoutForm` 提交订单信息。

这个练习做完，你对 React 日常开发就有感觉了。

## 8. 什么时候学 Next.js

当你能独立写：

- 商品列表。
- 购物车。
- 表单提交。
- 数据请求。
- 组件拆分。

就可以开始学 Next.js。

不要等 React 全部精通。

Next.js 主要解决：

- 多页面路由。
- SEO。
- 服务端渲染。
- 静态生成。
- API 接口。
- 部署。

如果你要做官网、SEO、支付接口，Next.js 很有价值。

## 9. 暂时不用急着学什么

初学阶段不要急：

- Redux。
- Zustand。
- React Query。
- React.memo。
- useCallback 优化。
- Fiber 原理。
- 大型架构。

这些不是没用，而是时机没到。

先把：

```text
组件
props
state
事件
列表
表单
useEffect
组件拆分
```

学扎实。

## 10. 推荐最终练习项目

### 项目 1：Todo List

功能：

- 新增任务。
- 删除任务。
- 完成任务。
- 过滤全部/已完成/未完成。
- 保存到 localStorage。

### 项目 2：商品购物车

功能：

- 商品列表。
- 加入购物车。
- 数量加减。
- 总价。
- 结算表单。

### 项目 3：后台列表页

功能：

- 请求列表。
- 搜索。
- 筛选状态。
- 新增。
- 编辑。
- 删除。

### 项目 4：小型点餐页

功能：

- 菜品分类。
- 菜品列表。
- 购物车。
- 结算。
- 提交订单。

这几个做完，再学 Next.js 会非常顺。
