# 手动验证记录：餐厅 Web 应用

## 首页

- [ ] 桌面端首页展示餐厅名称、餐饮定位、营业时间、地址、电话和主要入口。
- [ ] 移动端首页导航和主要入口不遮挡内容。
- [ ] “查看菜单”“预订座位”“开始点单”都能进入对应流程。

## 菜单

- [ ] 分类切换后菜品列表正确变化。
- [ ] 饮食标签筛选后结果正确变化。
- [ ] 无匹配结果时显示空状态，并可清除筛选。
- [ ] 不可售菜品显示“暂不可售”，且不能加入购物车。

## 预订

- [ ] 缺失姓名、联系方式、日期、时间或有效人数时阻止提交。
- [ ] 过去日期阻止提交。
- [ ] 有效预订提交后生成待确认请求。

## 点单

- [ ] 可售菜品可以加入购物车。
- [ ] 数量调整、移除菜品、小计和总计正确更新。
- [ ] 空购物车不能提交。
- [ ] 缺失姓名、联系方式或取餐方式不能提交。
- [ ] 有效点单提交后生成待确认订单。

## 后台

- [ ] 后台显示未认证不可生产使用的提示。
- [ ] 新增、编辑、删除、上下架菜品后，菜单侧能反映变化。
- [ ] 预订状态可以更新，并在列表中显示新状态。
- [ ] 订单状态可以更新，并在列表中显示新状态。

## 响应式与可访问性

- [ ] 桌面端主要页面无内容遮挡、无横向滚动、按钮文字不溢出。
- [ ] 移动端主要页面无内容遮挡、无横向滚动、按钮文字不溢出。
- [ ] 表单字段有标签，错误提示靠近相关字段。
- [ ] 主导航、表单和主要按钮可通过键盘访问。
- [ ] 状态变化通过文字或标签表达，不只依赖颜色。
