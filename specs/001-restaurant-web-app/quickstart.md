# 快速验证指南：餐厅 Web 应用

## 前置条件

- 项目依赖已安装。
- 开发服务器可以启动。
- 浏览器可以访问本地开发地址。

## 启动验证

1. 运行依赖安装命令。
2. 运行 lint、typecheck、test 和 build；若某项脚本尚未建立，记录为任务阶段待补充。
3. 启动开发服务器。
4. 打开本地地址，确认首页可以访问。

## 顾客首页验证

1. 打开首页。
2. 确认展示餐厅名称、餐饮定位、营业时间、地址、电话和主要入口。
3. 点击“查看菜单”“预订”“开始点单”，确认都能进入对应流程。
4. 缩小到移动端宽度，确认导航和主要入口不遮挡内容。

## 菜单验证

1. 打开菜单页。
2. 切换分类，确认菜品列表按分类变化。
3. 选择饮食标签，确认筛选结果立即变化。
4. 选择没有匹配结果的筛选组合，确认出现空状态。
5. 尝试将不可售菜品加入购物车，确认系统阻止操作。

## 预订验证

1. 打开预订表单。
2. 留空必填字段提交，确认系统提示错误。
3. 输入过去日期或无效人数提交，确认系统阻止提交。
4. 输入有效姓名、联系方式、日期、时间和人数提交。
5. 确认系统显示“待确认”性质的成功反馈。

## 点单/购物车验证

1. 从菜单加入多个可售菜品。
2. 调整数量并移除其中一个菜品，确认小计和总计正确更新。
3. 空购物车提交，确认系统阻止操作。
4. 填写顾客姓名、联系方式和取餐方式后提交。
5. 确认系统显示“待确认”性质的成功反馈。

## 后台验证

1. 打开后台入口。
2. 确认页面显示“开发占位/未认证不可生产使用”的提示。
3. 新增或编辑菜品，确认菜单侧能看到变化。
4. 将菜品标记为不可售，确认顾客侧不能加入购物车。
5. 更新预订和订单状态，确认列表与详情同步显示新状态。

## 性能与体验检查

1. 菜单分类切换、标签筛选和购物车数量调整应在 1 秒内出现可见反馈。
2. 移动端和桌面端主要页面不应出现横向滚动、文字重叠或按钮不可点击。
3. 表单错误必须靠近相关字段展示。
4. 主要操作必须可用键盘访问。
